🚀 Service Overview

Welcome to the GitHub Copilot API Proxy! This service provides an OpenAI-compatible API interface for GitHub Copilot, allowing you to integrate Copilot's AI capabilities into your applications.

🔑 Quick Start

  1. Get your API token: Authenticate with GitHub
  2. Use the API: Configure your OpenAI client to use {{ base_url }}/v1
  3. Start coding: Make requests to /v1/chat/completions

💻 Usage Examples

🐍 Python (OpenAI SDK)

import openai

client = openai.OpenAI(
    api_key="your-token-from-authentication",
    base_url="{{ base_url }}/v1"
)

response = client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "user", "content": "Write a Python function to calculate fibonacci"}
    ]
)

print(response.choices[0].message.content)

🌐 cURL

curl -X POST {{ base_url }}/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {"role": "user", "content": "Hello, GitHub Copilot!"}
    ]
  }'

🟨 JavaScript/Node.js

import OpenAI from 'openai';

const openai = new OpenAI({
    apiKey: 'your-token-from-authentication',
    baseURL: '{{ base_url }}/v1'
});

const completion = await openai.chat.completions.create({
    messages: [{ role: 'user', content: 'Generate a React component' }],
    model: 'gpt-4',
});

console.log(completion.choices[0].message);

🛡️ Security Features

  • ✅ OAuth2 authentication via GitHub
  • ✅ Secure token storage with encryption
  • ✅ Rate limiting and request validation
  • ✅ Unix domain socket communication
  • ✅ TLS termination via reverse proxy
  • ✅ Systemd security hardening

⚙️ Configuration

This service supports multiple deployment modes:

  • Unix Socket + Caddy: Recommended for production
  • Direct TLS: For standalone deployments
  • Development Mode: For local development

Configuration is managed via /etc/copilot-api-proxy/config.env
