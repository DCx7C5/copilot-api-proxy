🚀 Service Overview

Welcome to the OpenClaude Copilot Proxy — a production-ready TLS-encrypted proxy that lets OpenClaude, Claude Code, and other agents talk to GitHub Copilot, xAI Grok, and Anthropic Claude using a single OpenAI-compatible endpoint.

🔑 Quick Start

  1. Get your token: Authenticate →
  2. Configure client: point it at {{ base_url }}/v1
  3. Pick a model: gpt-4o, grok-3, claude-sonnet-4-6…

💻 Usage Examples

🐍 Python — OpenAI SDK

import openai

client = openai.OpenAI(
    api_key="cp-...",          # token from /login
    base_url="{{ base_url }}/v1"
)

# GitHub Copilot
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Write a Fibonacci function in Python"}]
)
print(response.choices[0].message.content)

🐍 Python — OpenClaude / Claude Code

import os
os.environ["ANTHROPIC_BASE_URL"] = "{{ base_url }}"
os.environ["ANTHROPIC_AUTH_TOKEN"] = "cp-..."   # token from /login

# Claude Code will automatically use the proxy

⚡ xAI Grok

client = openai.OpenAI(
    api_key="xai-...",         # your xAI key, or use server-configured key
    base_url="{{ base_url }}/v1"
)

response = client.chat.completions.create(
    model="grok-3",
    messages=[{"role": "user", "content": "Hello, Grok!"}]
)

🌐 cURL

curl -sk -X POST {{ base_url }}/v1/chat/completions \
  -H "Authorization: Bearer cp-..." \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"Hello!"}]}'

🟨 JavaScript / Node.js

import OpenAI from 'openai';

const client = new OpenAI({
    apiKey: 'cp-...',
    baseURL: '{{ base_url }}/v1'
});

const res = await client.chat.completions.create({
    model: 'gpt-4o',
    messages: [{ role: 'user', content: 'Generate a React component' }],
});
console.log(res.choices[0].message.content);

🛡️ Security Features

  • ✅ OAuth2 authentication via GitHub
  • ✅ GitHub Device Flow (no callback URL needed)
  • ✅ Personal Access Token (PAT) auth
  • ✅ Secure token storage with encryption
  • ✅ Rate limiting and request size validation
  • ✅ End-to-end TLS — own certificates, no Caddy
  • ✅ Systemd security hardening
  • ✅ Automatic expired-token cleanup

⚙️ Deployment Modes

  • 🔒 Direct TLS — own cert on :8443 (default)
  • 🔌 Unix Socket — for integration with external proxies
  • 🛠️ Development — HTTP on :8000, no TLS

Configuration: /etc/copilot-api-proxy/config.env

🔑 Auth Methods

  • GitHub OAuth · Device Flow · PAT
  • Twitter/X.com Bearer Token
